High-Speed FPGA Data Acquisition for RadioHound

The Goal

To design a high-speed data acquisition system to increase the RadioHound sensor's sampling rate beyond its current 48MHz limit. This was achieved by streaming ADC data directly through the BeagleV-Fire's FPGA fabric into DDR memory, minimizing CPU overhead.

System Architecture

The architecture creates a direct hardware path from the sensor input to system memory, using a custom Verilog module to pack data and a DMA controller to manage the high-speed transfer. Bare-metal C drivers provide precise, low-level control over the hardware.

Key Achievement & Validation

The core challenge was ensuring the DMA controller could correctly and efficiently access DDR memory through the FPGA fabric. I successfully validated the architecture by designing and executing a memory-to-memory DMA test, achieving a transfer rate of 400 MSPS (Mega Samples Per Second). This critical result proved the viability of the high-speed data path before integrating the final sensor hardware.
